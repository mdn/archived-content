JS_EnterCrossCompartmentCall has been removed in {{bug(786068)}}. Use {{jsapixref("JS_EnterCompartment")}} instead.

Enter a different compartment on the given context, so that objects in that compartment can be accessed.

Syntax

JSCrossCompartmentCall *
JS_EnterCrossCompartmentCall(JSContext *cx, JSObject *target);
Name Type Description
cx {{jsapixref("JSRuntime", "JSContext *")}} The context on which a cross-compartment call is needed.
target {{jsapixref("JSObject", "JSObject *")}}

The object in a different compartment to be accessed. This implicitly identifies the compartment to be entered.

Description

Every {{jsapixref("JSRuntime", "JSContext")}} has a current compartment. Any access to an object in a different compartment must be bracketed by calling JS_EnterCrossCompartmentCall and {{jsapixref("JS_LeaveCrossCompartmentCall")}}.

On success, JS_EnterCrossCompartmentCall returns a pointer to a JSCrossCompartmentCall object that must be passed to JS_LeaveCrossCompartmentCall. Otherwise it returns NULL.
