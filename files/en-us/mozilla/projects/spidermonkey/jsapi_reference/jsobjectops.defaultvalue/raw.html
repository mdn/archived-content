The JavaScript engine calls the JSObjectOps.defaultValue and {{jsapixref("JSClass", "JSClass.convert")}} callbacks to convert objects to primitive values.

Syntax

typedef JSBool (*JSConvertOp)(JSContext *cx, JSObject *obj, JSType type,
    jsval *vp);
Name Type Description
cx {{jsapixref("JSRuntime", "JSContext *")}} Pointer to the JS context in which the conversion is needed.
obj {{jsapixref("JSObject", "JSObject *")}} The object to be converted.
type {{jsapixref("JSType")}} The target type.
vp {{jsapixref("Jsval", "jsval *")}} Out parameter. On success, the callback must store the converted value here.

Description

The JSObjectOps.defaultValue callback corresponds to the [[DefaultValue]] method defined in ECMA 262-3 ยง8.6.2.6. Many expressions in the ECMAScript language, such as a+b, involve implicit conversion of the operands using [[DefaultValue]].

The default [[DefaultValue]] behavior for ordinary objects is as described in ES5 ยง8.12.8. It calls the JavaScript methods obj.valueOf() and/or obj.toString(). Provide a custom JSObjectOps.defaultValue callback to implement customized [[DefaultValue]] behavior.

The JSClass.convert callback should convert obj to the given type, returning JS_TRUE with the resulting value in *vp on success, and returning JS_FALSE on error or exception. On success, *vp must be a primitive value: per ES5 ยง8.12.8, every object "must ensure that its [[DefaultValue]] internal method can return only primitive values." Debug builds of SpiderMonkey will assert if a convert callback is successful but leaves *vp holding a primitive value.
