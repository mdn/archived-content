JSMarkOp is the type of the JSClass.mark callback in SpiderMonkey 1.7 and earlier. In SpiderMonkey 1.8, JSMarkOp will be deprecated. All new code using SpiderMonkey 1.8 or later should use a {{jsapixref("JSTraceOp")}} instead to ensure that the tracing APIs work properly.

Syntax

typedef uint32
(* JSMarkOp)(JSContext *cx, JSObject *obj, void *arg);
Name Type Description
cx {{jsapixref("JSRuntime", "JSContext *")}} The JS context in which the mark phase of garbage collection is occurring.
obj {{jsapixref("JSObject", "JSObject *")}} The object to be marked.
arg void * Opaque data which the mark hook must pass through to {{jsapixref("JS_MarkGCThing")}}.

Description

Call {{jsapixref("JS_MarkGCThing")}} for each GC thing directly reachable from obj. Use {{jsapixref("JSVAL_IS_GCTHING")}} to check whether a value needs to be marked and {{jsapixref("JSVAL_TO_GCTHING")}} to convert the jsval to a pointer.

JSClass hooks

JSClass offers the following hook:

  • The JavaScript engine calls the {{jsapixref("JSClass", "JSClass.mark")}} callback during the mark phase of garbage collection.
