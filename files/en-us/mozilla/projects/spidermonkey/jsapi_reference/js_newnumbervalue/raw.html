Convert a C floating-point number of type jsdouble to a jsval.

Syntax

JSBool
JS_NewNumberValue(JSContext *cx, jsdouble d, jsval *rval);
Name Type Description
cx {{jsapixref("JSRuntime", "JSContext *")}} The context in which to create the new number. {{ Jsapi-requires-request() }}
d {{jsapixref("jsdouble")}} The numeric value to convert.
rval {{jsapixref("jsval", "jsval *")}} Out parameter. On success, *rval receives a JavaScript number with the value d.

Description

JS_NewNumberValue converts a C floating-point number of type {{jsapixref("jsdouble")}} to {{jsapixref("jsval")}}, the type of JavaScript values.

The result is an integer jsval if d can be stored that way. Otherwise, a new floating-point jsval is created.

Some space for the new value may be allocated from the JS garbage collection heap. In this case, the value is subject to garbage collection until you protect against it using a local root, an object property, or the {{jsapixref("JS_AddRoot")}} function. This can be avoided by using the address of a rooted variable as rval.

On success, JS_NewNumberValue stores a numeric jsval in *rval and returns JS_TRUE. Otherwise it returns JS_FALSE and the value left in *rval is unspecified.
